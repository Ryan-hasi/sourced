# Sourced trademark policy

The code in this repository is MIT-licensed (see LICENSE). The **"Sourced"
name** and the **Sourced mark (the red dot lockup)** are not part of that
license. They identify the original project and its guarantees.

## You may, without asking

- State facts: "built on Sourced", "uses @sourcedhq/core", "compatible with
  the Sourced spec", with links to https://sourced.ink.
- Fork, modify and ship the code (MIT) — under a different name.
- Write about Sourced, screenshot it, benchmark it.

## You may not

- Name a fork, derivative, product or service "Sourced" or something
  confusingly similar, or use the mark for it.
- Imply endorsement by or affiliation with the Sourced project without
  written permission.
- Call an engine **"Sourced-conformant"** unless it passes the current
  @sourcedhq/conformance suite in full, unmodified. The suite is the sole
  yardstick for that claim.

## The ✓ Sourced mark

Products whose corroboration engine **passes the current
@sourcedhq/conformance suite, unmodified and in full,** may display the
✓ Sourced mark (badges: https://sourced.network/mark/) under these terms:

1. **Proof stays public.** The passing suite run (CI log or published JSON
   report) must be publicly reachable for the shipped version.
2. **Attribution is mandatory.** The badge or the claims "Sourced-conformant"
   / "Built on Sourced" must link to https://sourced.network, and the
   product's documentation must state the honest frame: verdicts mean
   *corroborated*, never *"true"*.
3. **Revocation.** Displaying the mark without a passing run, on a modified
   suite, or without attribution is a violation of this policy; we may say so
   publicly and revoke permission.

The code is MIT and free without any of this — the mark is what is earned.
